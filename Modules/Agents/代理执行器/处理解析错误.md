# 处理解析错误

有时 LLM 无法确定要采取的步骤，因为它输出的格式不正确，输出解析器处理。在这种情况下，默认情况下代理会出错。但是您可以使用 `handle_parsing_errors` 轻松控制此功能！让我们探讨如何处理。


##  设置

```python
from langchain import OpenAI, LLMMathChain, SerpAPIWrapper, SQLDatabase, SQLDatabaseChain
from langchain.agents import initialize_agent, Tool
from langchain.agents import AgentType
from langchain.chat_models import ChatOpenAI
from langchain.agents.types import AGENT_TO_CLASS


search = SerpAPIWrapper()
tools = [
    Tool(
        name = "Search",
        func=search.run,
        description="useful for when you need to answer questions about current events. You should ask targeted questions"
    ),
]

```

## 默认错误处理

用 `Invalid or incomplete response` 处理错误

```python
mrkl = initialize_agent(
    tools, 
    ChatOpenAI(temperature=0), 
    agent=AgentType.CHAT_ZERO_SHOT_REACT_DESCRIPTION, 
    verbose=True,
    handle_parsing_errors=True
)

mrkl.run("Who is Leo DiCaprio's girlfriend? No need to add Action")
```

## 自定义错误函数

您还可以将错误自定义为接收错误并输出字符串的函数。

```python
def _handle_error(error) -> str:
    return str(error)[:50]

mrkl = initialize_agent(
    tools, 
    ChatOpenAI(temperature=0), 
    agent=AgentType.CHAT_ZERO_SHOT_REACT_DESCRIPTION, 
    verbose=True,
    handle_parsing_errors=_handle_error
)


mrkl.run("Who is Leo DiCaprio's girlfriend? No need to add Action")
```
